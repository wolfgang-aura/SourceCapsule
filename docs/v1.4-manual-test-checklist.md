# SourceCapsule v1.4 Manual Chrome/X Test Checklist

Use this checklist with the Chrome profile that is already logged in to X. Automated tests cover
the pure capture/rendering engine and extension package; these checks verify X's current live DOM,
your session's media access, Chrome folder permissions, and the production share Worker.

## 1. Build and load the extension

Open PowerShell in the repository root and run:

```powershell
npm.cmd install
npm.cmd test
npm.cmd run lint
npm.cmd run format:check
npm.cmd run build:extension
Get-Content .\dist\sourcecapsule-extension\manifest.json
```

Confirm the manifest says `"version": "1.4.0"`, contains the production Worker host, and contains
neither `localhost` nor `127.0.0.1`.

In Chrome:

1. Open `chrome://extensions`.
2. Turn **Developer mode** on at the top right.
3. If an older unpacked SourceCapsule card points to a different folder, click **Remove**, then
   **Remove** again. If it already points to this repository's `dist` folder, click its circular
   **Reload** button instead.
4. Click **Load unpacked**.
5. Select the repository's `dist\sourcecapsule-extension` folder, then click **Select Folder**.
6. On the SourceCapsule card, confirm version **1.4.0** and no red error badge.
7. Open the Tampermonkey dashboard, find the SourceCapsule userscript, and turn only that script
   **off**. Leave Tampermonkey itself and unrelated scripts unchanged. This prevents duplicate
   SourceCapsule controls.
8. Refresh every X tab that was open before loading/reloading the extension (`Ctrl+R` in each).
9. Pin SourceCapsule: click Chrome's puzzle-piece icon, then the pin beside **SourceCapsule**.

If an extension failure occurs, open `chrome://extensions`, click **Errors** on the SourceCapsule
card, and save a screenshot plus the full first error. For a page failure, press `F12`, select
**Console**, type `SourceCapsule` in the filter, reproduce once, and copy the red error including
its stack trace.

## 2. Popup and preferences

Record PASS or FAIL beside each item.

- [ ] **P01 — Off X:** Open `https://example.com`, click the SourceCapsule toolbar icon. It says
  SourceCapsule runs on X, shows version 1.4.0, and **Open X** opens `https://x.com/`.
- [ ] **P02 — Supported X page:** On a refreshed X post page, click the icon. It says
  **Post or thread ready**, says quote/long-form/media recovery is active, and exposes all settings.
- [ ] **P03 — Refresh recovery:** Reload the extension at `chrome://extensions`, return to an X tab
  without refreshing it, and open the popup. It explicitly asks for a refresh. Refresh X; the ready
  state returns.
- [ ] **P04 — Settings persistence:** Select **Flat**, **Lean Markdown + media**, and turn
  **Floating button** on. Close/reopen the popup, then fully exit and restart Chrome. All three
  settings remain selected. Restore **By date**, **Full archive**, and your preferred floating
  button state afterward.
- [ ] **P05 — Change export folder:** Click **Change export folder**. On the X page, click
  **Choose folder** in the SourceCapsule prompt, select a disposable test folder, and click
  **Select Folder**. No folder picker may fail silently. Run this case in Google Chrome. Brave
  intentionally disables the native File System Access API; in Brave, PASS means this control is
  disabled with a clear message that library captures download as ZIP files instead.
- [ ] **P06 — Links:** **Help / source** and **Privacy** open the repository and SECURITY page.
- [ ] **P07 — Reply-context toggle:** Un-check **Include reply context**, save any reply
  post, and confirm no "In reply to" card appears. Re-check it, export the same reply, and
  confirm the card is back. The Tampermonkey menu command **Reply context** must show the
  same state.
- [ ] **P08 — Popup polish and keyboard access:** Check the popup in light and dark Chrome themes.
  PASS if no text clips at 100% and 125% display scale, every setting shows **Setting saved**, the
  selected segmented options remain obvious, and `Tab` reaches every radio/switch/button/link with
  a visible focus ring. On an X post, open the SourceCapsule action menu, use arrow keys, then press
  `Escape`; focus must return to the ellipsis button.

Failure evidence: popup screenshot, SourceCapsule extension error, and page-console error.

## 4. Full and Lean library saves plus receipts

1. In the popup choose **By date** and **Full archive**.
2. Save a media-bearing post with **Save to library**.
3. PASS if the receipt remains until **Done** or `Escape`, its location exists, and the folder has:
   the `.html`, `.llm.md`, `README.txt`, and matching `media\` files. Compare posts, captured/missing
   images, offline videos, posters, incomplete media, and recovered Notes to the files and HTML.
4. In the receipt click **Copy for AI**, open Notepad (`Win+R`, type `notepad`, press Enter), press
   `Ctrl+V`, and confirm clean Markdown—not HTML—was pasted.
5. Repeat with popup settings **Flat** and **Lean Markdown + media**.
6. PASS if the flat post folder has `.llm.md`, `README.txt`, and `media\`, has no `.html`, and the
   receipt reports **0 videos preserved offline**. Images and video poster stills must still exist.
7. Trigger a save with known unavailable media. PASS if the receipt stays visible, shows a warning,
   and expanded details explain the missing/incomplete item.
8. With a receipt open, press `Escape`. PASS if it closes and the saved files remain.

Failure evidence: receipt screenshot, entire saved capsule folder (zip it in Explorer if needed),
the X URL, and the filtered SourceCapsule console error.

## 5. Production sharing and hosted-only CTA

Use a small capture below 25 MB. Sharing creates an unlisted public link, so use content you are
comfortable making accessible to anyone with that link.

1. From a local-save receipt click **Create AI readable link**.
2. Confirm no upload starts before the confirmation form. Choose **1 day**, enter a disposable note
   and tag, then click **Save + create link**.
3. PASS if progress shows each upload, the local-success receipt remains the single result surface,
   and it displays the public link without losing the saved-folder result.
4. Open the saved local folder and confirm `AI_LINK.txt` exists with the public link, Markdown link,
   source URL, expiry, and any missing/incomplete media count.
5. Open **SourceCapsule: Recent AI readable links** from the userscript manager menu. PASS if the
   new link appears, old expired links are greyed out, and Copy/Open/Copy Markdown/Delete controls
   are visible.
6. Open the public link. Confirm text, quote cards, polls, images/posters, `noindex` behavior, and the
   footer copy:
   `Captured with SourceCapsule` and
   `Preserve complete X threads for AI and offline reading.`
7. Click **Install SourceCapsule**. Until the Store URL is supplied, it must open the repository's
   installation section—not an invented Web Store listing.
8. If the public URL is `https://host/c/ID`, open `https://host/c/ID.md`,
   `https://host/c/ID/manifest.json`, and one `https://host/c/ID/media/...` URL named by the
   manifest. Each must load. Full raw video must not be uploaded.
9. Open the local `.html` saved before sharing and search for `Install SourceCapsule`. PASS only if
   there is no match; the acquisition footer is hosted-only.
10. Repeat once with **7 days** or **30 days**. In DevTools **Network**, select the successful
   `POST /api/capsules` request, open **Response**, and confirm its `expiresAt` matches the selection
   within normal clock tolerance.
11. Optional failure-safety check: disconnect the network after a local save, click
   **Create AI readable link**, confirm, and let it fail. PASS if the receipt says the local save is
   safe and the local folder remains intact. Reconnect afterward.

Failure evidence: public capsule URL, expiry selected, receipt screenshot, failing endpoint and HTTP
status from DevTools **Network**, `/manifest.json`, and the SourceCapsule console error. Do not send
upload/delete tokens.

## 6. Strict export gate (v1.4)

The gate is on by default. Its job is to make silent broken exports impossible.

- [ ] **G01 — Clean export is silent:** Save a post/thread that captures cleanly (all images
  inline, no dead quotes). PASS if the export downloads without any modal appearing.
- [ ] **G02 — Gate blocks a known-broken export:** Find a post that quotes an account whose
  content can no longer be captured (a suspended account, or a private account you don't
  follow). Save it. PASS if a red modal appears before the download listing the incomplete
  item(s), and the download does NOT proceed while the modal is open. FAIL if the modal is
  missing or the file downloads first and then the modal appears.
- [ ] **G03 — Cancel path leaves nothing behind:** In the modal, click **Cancel export**.
  PASS if the modal closes, a toast explains strict mode is on, and no partial download
  appears in your Downloads folder.
- [ ] **G04 — Ship-it-anyway path works:** Reproduce G02, then click **Ship it anyway**.
  PASS if the download proceeds and the file matches the pre-gate behavior (may contain
  the dead-end card you were warned about — that's the point of ship-it-anyway).
- [ ] **G05 — Diagnostic bundle copy:** Reproduce G02, click **Copy diagnostic bundle**.
  Paste into Notepad. PASS if the pasted text is valid JSON containing `generator`,
  `verdict: "incomplete"`, `counts`, `blockers[]` naming the affected handle(s), and a
  `modelSkeleton` block. FAIL if it contains base64 media (`data:image/...`) — the bundle
  must not leak bytes.
- [ ] **G06 — Toggle strict mode off via popup:** Open the extension popup and un-check
  **Strict export (block incomplete)**. Repeat G02. PASS if the export downloads silently
  with no modal (old behavior). Re-check the toggle when done.
- [ ] **G07 — Toggle strict mode off via manager menu:** In Tampermonkey / Violentmonkey's
  menu, click **SourceCapsule: Strict export - on (click to switch)** and confirm the label
  flips to `off`. Same-domain popup must reflect the new state.
- [ ] **G08 — Auto-repair runs before the modal:** Save a media-heavy thread (20+ images)
  on a throttled connection (DevTools Network → Slow 3G helps reproduce). If anything
  fails on the first pass, PASS when you see a `Retrying N incomplete item(s)...` toast
  and the export then completes with NO modal (the repair round recovered the failures).
  The modal appearing immediately with no retry toast in between is a FAIL.
- [ ] **G09 — Modal only lists what repair could not fix:** Reproduce G02 (a genuinely
  dead quote — suspended/private account). PASS if the retry toast appears first, is
  followed by the modal after a few seconds, and the modal lists only the genuinely
  unfixable item(s). The console should show `strict-gate quote repair failed for <id>`.
- [ ] **G10 — Retry recovery button in the modal:** With the modal open (G02/G09), click
  **Retry recovery**. PASS if the buttons grey out while a `Retrying incomplete
  items...` status shows, then either (a) the export proceeds with no further clicks
  because everything recovered, or (b) the status reads `Still incomplete: N item(s)
  could not be recovered.` and the blocker list/count refreshes to match. Cancel and
  Ship it anyway must still work after a failed retry.

## 7. Thread menu escape hatch (v1.4)

- [ ] **T01 — Detected thread quick action:** Open a same-author thread page where auto-
  detection identifies the thread. PASS if the button says **Save thread** and quick-click
  captures all posts.
- [ ] **T02 — Escape hatch is always present on focused post:** Open any post's status page.
  Click the drop-down (⋯) beside its button. PASS if the menu's first item is **Save full
  thread**, regardless of whether the button label says "Save thread" or "Save post".
- [ ] **T03 — Escape hatch actually captures the thread:** On a focused post whose button
  says **Save post** but you know is a thread, choose **Save full thread** from the drop-
  down. PASS if the receipt reports more than 1 post captured (the scroll-and-load path
  found the follow-ups even though auto-detection missed them at menu-open time).
- [ ] **T04 — Continuation posts stay post-only:** On a same-author thread, open the drop-
  down of a *reply* post (not the focused post at the top of the URL). PASS if that menu
  does NOT contain **Save full thread** — thread mode belongs to the focused post only.


## 8. Quote source recovery (v1.4)

- [ ] **Q01 — Captured refs beat DOM loss:** Find a thread whose visible quote cards do not
  have `/status/` anchors in the DOM (rare but observable on older cached posts). Save the
  thread. PASS if the exported HTML's quote cards all have real permalinks in their
  `View on X →` links, not `x.com/{handle}` profile fallbacks.
- [ ] **Q02 — Profile fallback when everything misses:** If Q01 is impossible to find,
  temporarily set `CONFIG.useSyndication=false` in DevTools (`window` is not exposed; you
  can toggle it in a copy of the userscript if needed). Save a thread with quotes. PASS if
  quote cards missing exact URLs link to the author's profile with a `View @handle on X →`
  label and `data-xa-source-fallback="author-profile"` attribute — never a "Source URL
  unavailable" notice.
- [ ] **Q03 — Shared exports serve media with Content-Length:** Create an AI readable link
  from any export with images. Open DevTools **Network** on the public capsule URL. Click
  an `image-*.jpg` request. PASS if the response headers include `Content-Length` matching
  the response size. This is what makes Slack, Discord, and Twitter link previews render.
- [ ] **Q04 — Single post recovers a never-rendered image:** Open an image post's status
  page and export it IMMEDIATELY after the page paints (before the image finishes
  loading; Slow 3G throttling makes this easy). PASS if the exported HTML contains the
  image anyway — the console shows `post syndication pass: recovered 1 item(s) the DOM
  missed` when the recovery actually fired.
- [ ] **Q05 — Banned/deleted quoted post gets an honest note:** Export a post whose quote
  shows X's "This Post is unavailable" box (e.g. a quote of a banned account). PASS if
  the receipt adds a **Quoted posts gone on X: 1** row while the status line still says
  Complete, no strict-gate modal appears (nothing was capturable), and the exported HTML
  and Markdown both contain a "Quoted post unavailable on X" note quoting X's message.
- [ ] **Q06 — Poll/link metadata survives quote rebuild:** Export a post quoting a poll or a rich
  external-link card. PASS if syndication corrects the quote author/text/media without removing the
  poll choices, expanded destination URL, card title, or thumbnail that X rendered in the DOM.

## 9. Folder permission recovery after Chrome restart

1. Complete one successful Full library save and note the chosen root folder.
2. Exit Chrome completely: menu **⋮** > **Exit**. Reopen Chrome and X.
3. Save another post without using **Change export folder**.
4. If Chrome asks to restore write access, click **Allow**. PASS if SourceCapsule reuses the same
   root and the receipt points to the new post folder. A one-session permission confirmation is
   acceptable; silently changing folders or cancelling the export is not.

Failure evidence: permission prompt screenshot, receipt/toast screenshot, chosen root path, and
page-console error.
