# SourceCapsule v1.3 Manual Chrome/X Test Checklist

Use this checklist with the Chrome profile that is already logged in to X. Automated tests cover
the pure capture/rendering engine and extension package; these checks verify X's current live DOM,
your session's media access, Chrome folder permissions, and the production share Worker.

## 1. Build and load the extension

Open PowerShell and run:

```powershell
cd C:\Users\cheon\Desktop\Projects\eXportArticle
npm.cmd install
npm.cmd test
npm.cmd run lint
npm.cmd run format:check
npm.cmd run build:extension
Get-Content .\dist\sourcecapsule-extension\manifest.json
```

Confirm the manifest says `"version": "1.3.0"`, contains the production Worker host, and contains
neither `localhost` nor `127.0.0.1`.

In Chrome:

1. Open `chrome://extensions`.
2. Turn **Developer mode** on at the top right.
3. If an older unpacked SourceCapsule card points to a different folder, click **Remove**, then
   **Remove** again. If it already points to this repository's `dist` folder, click its circular
   **Reload** button instead.
4. Click **Load unpacked**.
5. Select
   `C:\Users\cheon\Desktop\Projects\eXportArticle\dist\sourcecapsule-extension`, then click
   **Select Folder**.
6. On the SourceCapsule card, confirm version **1.3.0** and no red error badge.
7. Open the Tampermonkey dashboard, find the SourceCapsule userscript, and turn only that script
   **off**. Leave Tampermonkey itself and unrelated scripts unchanged. This prevents duplicate
   SourceCapsule controls.
8. Refresh every X tab that was open before loading/reloading the extension (`Ctrl+R` in each).
9. Pin SourceCapsule: click Chrome's puzzle-piece icon, then the pin beside **SourceCapsule**.

If an extension failure occurs, open `chrome://extensions`, click **Errors** on the SourceCapsule
card, and save a screenshot plus the full first error. For a page failure, press `F12`, select
**Console**, type `SourceCapsule` in the filter, reproduce once, and copy the red error including
its stack trace.

## 2. Popup and preferences

Record PASS or FAIL beside each item.

- [ ] **P01 — Off X:** Open `https://example.com`, click the SourceCapsule toolbar icon. It says
  SourceCapsule runs on X, shows version 1.3.0, and **Open X** opens `https://x.com/`.
- [ ] **P02 — Supported X page:** On a refreshed X post page, click the icon. It says
  `SourceCapsule is ready on this page.` and exposes all settings.
- [ ] **P03 — Refresh recovery:** Reload the extension at `chrome://extensions`, return to an X tab
  without refreshing it, and open the popup. It explicitly asks for a refresh. Refresh X; the ready
  state returns.
- [ ] **P04 — Settings persistence:** Select **Flat**, **Lean Markdown + media**, and turn
  **Floating button** on. Close/reopen the popup, then fully exit and restart Chrome. All three
  settings remain selected. Restore **By date**, **Full archive**, and your preferred floating
  button state afterward.
- [ ] **P05 — Change export folder:** Click **Change export folder**. On the X page, click
  **Choose folder** in the SourceCapsule prompt, select a disposable test folder, and click
  **Select Folder**. No folder picker may fail silently. Run this case in Google Chrome. Brave
  intentionally disables the native File System Access API; in Brave, PASS means this control is
  disabled with a clear message that library captures download as ZIP files instead.
- [ ] **P06 — Links:** **Help / source** and **Privacy** open the repository and SECURITY page.

Failure evidence: popup screenshot, SourceCapsule extension error, and page-console error.

## 3. Capture matrix on live X

Choose public examples you can legally retain. Before each case, note its X URL. Use the inline
**Save** control or the SourceCapsule action menu. Wait for progress to finish; do not close the tab
while `[n/total]` or media progress is changing.

| ID | Case | PASS criteria | Return on FAIL |
| --- | --- | --- | --- |
| C01 | Single text post | Receipt says 1 post; HTML/Markdown contains the complete text and correct author/source. | X URL, receipt screenshot, saved folder |
| C02 | Same-author thread | Receipt post count matches the intended visible thread; recommendations/replies are absent; post markers and URLs are ordered. | X URL, expected vs actual post count, saved folder |
| C03 | X Article | Title, author, headings, lists, links, body images, and embedded posts keep their order. | X URL, screenshot of wrong section, saved folder |
| C04 | Long-form Note thread | Full Note text is present when X delivered it; receipt/manifest counts recovered Notes. Otherwise a visible truncation warning is present—never a silent preview. | X URL, receipt, console error, saved HTML |
| C05 | Direct quote | Quote author, text, timestamp/source, poll/media, and attribution are inside the quote card only. | X URL and screenshot of source vs archive |
| C06 | Nested quote | Each nesting level appears once; no quote text, poll, or image leaks into the outer post. | X URL, archive screenshot, saved folder |
| C07 | Image gallery | Receipt captured/missing counts match visible images; each captured image opens in the offline HTML lightbox and exists under `media\`. | X URL, receipt, missing filenames |
| C08 | Downloadable MP4 | Full archive HTML plays the video with Wi-Fi disabled; receipt reports one offline video. Lean mode must report zero offline videos because it intentionally stores only the poster/source link. | X URL, receipt, browser console error, saved HTML |
| C09 | HLS/poster fallback | Receipt reports incomplete media; archive shows the poster and **Watch video on X**, and never claims the full video is offline. | X URL, receipt, saved HTML |
| C10 | Open poll | Every visible choice appears; hidden results are labelled unavailable and no zero/percentage is invented. | X URL, source + archive screenshots, embedded manifest |
| C11 | Closed poll | Visible percentages/vote total/status appear in HTML, Markdown, and manifest. | X URL, source + archive screenshots, manifest |
| C12 | Poll inside quote | Poll appears only under the quoted post; the outer post does not duplicate it. | X URL, archive screenshot, manifest |

To inspect a local HTML manifest, open the saved `.html` in Notepad, press `Ctrl+F`, search for
`sourcecapsule-debug`, and copy the JSON inside that script element. Do not edit the file.

## 4. Full and Lean library saves plus receipts

1. In the popup choose **By date** and **Full archive**.
2. Save a media-bearing post with **Save to library**.
3. PASS if the receipt remains until **Done** or `Escape`, its location exists, and the folder has:
   the `.html`, `.llm.md`, `README.txt`, and matching `media\` files. Compare posts, captured/missing
   images, offline videos, posters, incomplete media, and recovered Notes to the files and HTML.
4. In the receipt click **Copy for AI**, open Notepad (`Win+R`, type `notepad`, press Enter), press
   `Ctrl+V`, and confirm clean Markdown—not HTML—was pasted.
5. Repeat with popup settings **Flat** and **Lean Markdown + media**.
6. PASS if the flat post folder has `.llm.md`, `README.txt`, and `media\`, has no `.html`, and the
   receipt reports **0 videos preserved offline**. Images and video poster stills must still exist.
7. Trigger a save with known unavailable media. PASS if the receipt stays visible, shows a warning,
   and expanded details explain the missing/incomplete item.
8. With a receipt open, press `Escape`. PASS if it closes and the saved files remain.

Failure evidence: receipt screenshot, entire saved capsule folder (zip it in Explorer if needed),
the X URL, and the filtered SourceCapsule console error.

## 5. Production sharing and hosted-only CTA

Use a small capture below 25 MB. Sharing creates an unlisted public link, so use content you are
comfortable making accessible to anyone with that link.

1. From a local-save receipt click **Create share link**.
2. Confirm no upload starts before the confirmation form. Choose **1 day**, enter a disposable note
   and tag, then click **Save + create link**.
3. PASS if progress shows each upload, the local-success receipt remains the single result surface,
   and it displays the public link without losing the saved-folder result.
4. Open the public link. Confirm text, quote cards, polls, images/posters, `noindex` behavior, and the
   footer copy:
   `Captured with SourceCapsule` and
   `Preserve complete X threads for AI and offline reading.`
5. Click **Install SourceCapsule**. Until the Store URL is supplied, it must open the repository's
   installation section—not an invented Web Store listing.
6. If the public URL is `https://host/c/ID`, open `https://host/c/ID.md`,
   `https://host/c/ID/manifest.json`, and one `https://host/c/ID/media/...` URL named by the
   manifest. Each must load. Full raw video must not be uploaded.
7. Open the local `.html` saved before sharing and search for `Install SourceCapsule`. PASS only if
   there is no match; the acquisition footer is hosted-only.
8. Repeat once with **7 days** or **30 days**. In DevTools **Network**, select the successful
   `POST /api/capsules` request, open **Response**, and confirm its `expiresAt` matches the selection
   within normal clock tolerance.
9. Optional failure-safety check: disconnect the network after a local save, click **Create share
   link**, confirm, and let it fail. PASS if the receipt says the local save is safe and the local
   folder remains intact. Reconnect afterward.

Failure evidence: public capsule URL, expiry selected, receipt screenshot, failing endpoint and HTTP
status from DevTools **Network**, `/manifest.json`, and the SourceCapsule console error. Do not send
upload/delete tokens.

## 6. Folder permission recovery after Chrome restart

1. Complete one successful Full library save and note the chosen root folder.
2. Exit Chrome completely: menu **⋮** > **Exit**. Reopen Chrome and X.
3. Save another post without using **Change export folder**.
4. If Chrome asks to restore write access, click **Allow**. PASS if SourceCapsule reuses the same
   root and the receipt points to the new post folder. A one-session permission confirmation is
   acceptable; silently changing folders or cancelling the export is not.

Failure evidence: permission prompt screenshot, receipt/toast screenshot, chosen root path, and
page-console error.
