## Defects found and fixed

1. The old `Download HTML + Markdown` action created two loose sibling downloads. It now reads
   `Download ZIP (HTML + Markdown)` and emits one archive containing both companions. Markdown
   describes the ZIP relationship honestly. Commit: `54d8489`.
2. Ordinary-post Markdown incorrectly said media was attached to the `main article`. Root media is
   now labelled `main post` for posts and `main article` for Articles. Commit: `6aa2edf`.

Both fixes passed `npm.cmd test`, `npm.cmd run lint`, and `npm.cmd run format:check`. The production
extension package was rebuilt and its userscript SHA-256 matches the source:
`E9E4DDCC239CE7BF9F955224C23D4F953434F44DC95D7EFE3EA91F384CC53B11`.
